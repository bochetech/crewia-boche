📋 Backlog Épicas — Desglose Técnico

📦 Épica: Integración MultiVende (F1-106)

Problema: Quiebres de stock frecuentes por desincronización manual inventario 3PL.

Solución: API REST bidireccional (Shopify ↔ MultiVende) con webhook de actualización en tiempo real.

Historias de Usuario:

  • [US-001] Como operador, quiero que el stock se sincronice automáticamente cada 15 min para evitar sobreventa.
  • [US-002] Como desarrollador, quiero logs de errores de sincronización para debugging rápido.
  • [US-003] Como cliente, quiero ver stock real disponible antes de comprar para evitar cancelaciones.

🛠️ Prompts para Desarrollo: 

### Prompt 1: Diseño de API
"Diseña un adaptador BFF en Node.js (NestJS) que:
1. Expone endpoint POST /webhook/stock-update
2. Recibe payload de MultiVende con SKU, cantidad, warehouse
3. Actualiza Shopify Inventory API con retry exponencial
4. Almacena log en PostgreSQL para auditoría
5. Emite evento a Redis pub/sub para invalidar cache

Stack: NestJS, TypeScript, Bull (queues), PostgreSQL, Redis"

### Prompt 2: Manejo de Errores
"Implementa estrategia de resilencia para API MultiVende:
- Circuit breaker (Polly pattern): si 5 fallos consecutivos → pausar 30s
- Dead letter queue para updates fallidos
- Alertas Slack si tasa error > 5% en 1h
- Dashboard Grafana con métricas: latencia p95, tasa éxito, backlog queue"

### Prompt 3: Testing
"Genera suite de tests E2E con Jest:
1. Mock API MultiVende con diferentes escenarios (200, 400, 500, timeout)
2. Validar que Shopify se actualiza correctamente
3. Validar que errores se loguean sin crashear
4. Test de carga: 1000 webhooks/min simultáneos"

Historias de Usuario:

  • [US-005] Como viña, quiero gestionar disponibilidad de tours desde panel admin unificado.
  • [US-006] Como sistema, quiero enviar confirmación automática por email con QR code para check-in.

🛠️ Prompts para Desarrollo: 

### Prompt 1: Custom Shopify App
"Crea Shopify app con Remix (TypeScript) que:
1. Añade metafield 'bookable' a productos (true/false)
2. Si bookable=true → muestra calendario disponibilidad
3. Al agregar al carrito → valida cupo disponible (fetch a DB interna)
4. Genera line_item con atributos: fecha, hora, num_personas
5. Post-checkout → crea reserva en DB + envía email confirmación"

### Prompt 2: Calendario Disponibilidad
"Implementa sistema de slots de disponibilidad:
- DB: PostgreSQL con tabla 'tour_slots' (tour_id, date, time, capacity, booked)
- API GraphQL para frontend: query availableSlots(tourId, startDate, endDate)
- Optimización: cache Redis con TTL 5 min para fechas populares
- Validación: lock pesimista en DB al reservar (prevenir overbooking)"

### Prompt 3: Migración de Datos
"Script Python para migrar historial de Bókun a Shopify:
1. Exportar reservas desde Bókun API (últimos 2 años)
2. Transformar a formato Shopify metafields
3. Importar con Shopify Admin API (bulk operations)
4. Validar: comparar totales antes/después, reportar discrepancias"

Problema: Competidores (Vilaport, Vinos del Mundo) ajustan precios dinámicamente; Descorcha pierde ventas por no reaccionar a tiempo.

Solución: Scraper automático + alertas + sugerencias de ajuste de precio.

Historias de Usuario:

  • [US-007] Como pricing manager, quiero recibir alerta Slack cuando competidor baje precio >10% en SKU clave.
  • [US-008] Como sistema, quiero scrapear precios diariamente de 5 competidores principales.
  • [US-009] Como dashboard, quiero visualizar comparativa precio vs competencia en tiempo real.

🛠️ Prompts para Desarrollo: 

### Prompt 1: Web Scraper
"Implementa scraper Python (Scrapy + Playwright) que:
1. Navega sitios: vilaport.cl, vinosdelmundo.cl, winelovers.cl
2. Extrae: producto, precio, stock, descuentos (maneja JS dinámico)
3. Almacena en PostgreSQL: competitor_prices (sku, competitor, price, date)
4. Ejecuta diario vía cron (AWS Lambda + EventBridge)
5. Resiliencia: retries, rotate user-agents, manejo CAPTCHAs (2Captcha API)"

### Prompt 2: Sistema de Alertas
"Crea servicio Node.js que:
1. Compara precios Descorcha vs competidores cada 6h
2. Calcula delta porcentual para top 100 SKUs
3. Si delta > 10% → POST a Slack webhook con: SKU, precio_nuestro, precio_competidor, sugerencia
4. Almacena historial alertas para análisis tendencias"

### Prompt 3: Dashboard Comparativo
"Desarrolla dashboard React (Recharts) con:
- Gráfico línea: evolución precio producto vs 3 competidores (últimos 30 días)
- Tabla: SKUs donde estamos >15% más caros (ordenado por volumen ventas)
- Filtros: categoría, rango precio, competidor
- Backend: API FastAPI que sirve datos de PostgreSQL"
